核心设计原则

📐 代码做计算，LLM 做解释

所有数值计算（负荷预测、潮流仿真、投资评分）由 Python 确定性代码完成。LLM 只负责读取结构化结果，生成面向人员的文字说明。彻底规避弱模型计算出错的问题。

🔒 规则做过滤，模型做描述

布点约束（距离范围、禁建区域）由代码强制过滤；接线方案建议文字、投资策略匹配由 LLM 参与。关键判断有规则兜底，LLM 不可用时系统自动降级。

🖱️ 自动为主，人工兜底

每个关键节点（布点位置、接线方式）保留人工拖拽修正入口。AI 提供 Top3 建议，规划人员确认后流转，不替代专业判断。

🏢 全程内网离线

数据存储在本地 SQLite，嵌入模型本地运行，地图使用离线瓦片包。仅 LLM 推理依赖内网 API；API 不可用时降级为纯计算模式，不中断流程。

进程通信架构

// Electron 三层通信链路

Vue 渲染进程
  ↓ ipcRenderer.invoke()       // 触发计算 / 爬取 / LLM
Electron 主进程（IPC 总线）
  ↓ child_process.spawn()     // 调用 Python 计算引擎
Python 脚本（stdout JSON）    // 纯标准库，零额外安装
  ↓ fetch()（主进程发出）      // 绕过 CORS，直接调内网接口
内网 LLM API                  // DeepSeek R1 / Qwen
ℹ️
关键设计：所有数据库操作由 Python 负责。Python 标准库内置 sqlite3，零额外安装。Electron 主进程不直接操作数据库，彻底避免 better-sqlite3 等原生模块在内网的编译问题。

数据采集层（RPA）

系统数据内容频率策略优先级
计量自动化 台区日负载率、分时负荷曲线 每日 增量（只拉取新增日期） P0
GIS 设备中心 台区经纬度、供电范围、线路拓扑 每周 全量（台区数量稳定） P0
电网管理平台 资产台账：配变容量、线路型号、投运年份 每周 全量 diff P0
营销系统 报装负荷：已报装容量、意向报装、用户类型 每日 增量 P1
电压监测系统 低电压事件：电压值、发生时间、持续时长 每日 增量 P1

数据内存管理策略

负荷数据滚动窗口

近 2 年保留日粒度数据；更早的自动聚合为月均值并删除细粒度记录。每月执行一次压缩，PRAGMA vacuum 回收空间。

增量爬取逻辑

每次爬取前读取 last_sync_date，只拉取新增数据。UPSERT 写入（主键冲突则更新），避免重复数据。

RPA Preload 标准模式

监听 IPC 事件 → 等待页面就绪 → 抓包确认接口 → fetch 调接口 → JSON 写 SQLite → 通知完成。

非标浏览器应对

内网自研浏览器优先找数据 API；无接口时读 DOM 表格；有签名时复用页面已有签名函数；HttpOnly Cookie 改用 Electron session API 读取。

多智能体设计

四个智能体链式工作，每个智能体的输出作为下一个的输入。Python 负责所有计算，LLM 只负责文字。

AGENT 01 · 负荷预测
判定是否需要新建台区
代码 年均增长率 × N年 + 报装负荷（含折算系数）= 预测总负荷，计算届时负载率
代码 负载率 > 80% → 标记需新建，输出建议分割比例
LLM 解读预测结论，100 字以内，面向基层班组通俗描述
AGENT 02 · 布点方案
推荐 Top3 候选新建位置
代码 GIS 空间筛选（200-500m 半径）+ 硬约束过滤（禁建区、最近距离）
代码 综合评分排序（距离 40% + 均衡 30% + 费用 30%）→ Top3
LLM 读取候选点信息，建议 π接 / T接 及理由，描述接火解火位置
AGENT 03 · 仿真校验
验证方案电气合规性
代码 前推回代法（或静态查表过渡）计算各节点电压偏差、各线段负载率
代码 判断 π接 vs T接 是否满足：负载率 < 80%，电压偏差 < ±7%
LLM 读计算结果 JSON，指出越限节点，给出修改建议
AGENT 04 · 投资评估
多方案技术经济比选
代码 技术指标（电压改善 30%、负载改善 25%、供电可靠性 20%、均衡度 25%）× 权重评分
代码 经济指标（建设投资 40%、回收期 35%、单位费用 25%）× 权重评分
LLM 生成最优方案推荐报告（150 字），结合知识库检索的投资策略

π接 vs T接 判断规则

接线方式适用场景仿真拓扑变化代码判断条件
π接 原线路负载率 > 70%，或末端电压偏差 > 5% 原线路切断，新台串入中间，两段独立供电 current_load_rate > 70
T接 原线路较轻（<50%），新增负荷占比 < 30% 原线路不断，新台以分支挂接，主线不卸载 current_load_rate <= 50 且 dist_m > 300

方案 A · 静态查表法（当前实现）

适合快速验证，工程估算精度

基于导线型号查额定载流量，用电流公式估算负载率；用线路电阻×电流估算电压降。不需要完整拓扑数据。

所需数据：导线型号、线路长度、预测负荷。

精度：工程估算级，约 ±15%，可用于方案初筛。

已实现 simulate.py
方案 B · 前推回代法（完整实现，TBD）

工程级精度，需要完整拓扑

所需数据：节点拓扑（节点+线路）、各台区负荷、线路电阻。依赖 GIS 数据格式确认。

精度：工程计算级，满足规划要求。

LLM 使用策略（针对弱模型优化）

ℹ️
内网 LLM（DeepSeek R1 / Qwen）适合基础文字生成，不适合复杂数值计算和多步推理。本系统的设计原则是：把 LLM 当"写手"，不当"计算器"。

正确 vs 错误的 Prompt 设计

❌ 错误：让 LLM 计算
"台区A负荷120kW，增长率2%，
5年后负荷是多少？是否需要
新建台区？"

问题：弱模型可能算错，且无法验证。

✅ 正确：代码算完再给 LLM
"计算结果：当前负载率75%，
5年预测负载率92.5%，超过80%
预警线，建议新建台区。
请用100字通俗描述原因。"

LLM 只做解释，结果由代码保证。

System Prompt 核心内容（所有智能体共用）

你是南方电网配网规划助手。职责：根据系统提供的计算结果生成简洁规划建议。

背景知识：
- 重过载：台区配变负载率（负荷/容量×100%）超过 80%
- 低电压：用户端电压偏差低于标称电压（220V）的 -7%，即低于 204.6V
- π接：在现有线路切断，新台区串联接入，形成两段独立供电
- T接：在现有线路节点引出分支，新台区并联接入，原线路不断

注意：所有数值已由系统计算完成，请勿自行计算或修改数字。

LLM 不可用时的降级策略

// electron/index.ts - LLM 调用失败时自动降级
try {
  const res = await fetch(llmUrl, { signal: AbortSignal.timeout(30_000) })
  return { success: true, text: data.choices[0].message.content }
} catch {
  // 降级：返回兜底文字，不中断流程
  return { success: false, text: '[LLM 暂不可用，计算结果见上方数据]' }
}

技术选型

层次技术理由内网安装方式
桌面壳 Electron 29 复用现有 RPA 工具经验；访问 Node.js API；Cookie 注入 含在 node_modules，整体打包带入
前端框架 Vue 3 + TypeScript 与现有工具一致，复用组件 同上
UI 组件 Element Plus 成熟的表单/表格组件 同上
数据库 SQLite（Python 标准库） 零额外安装；Python 内置 sqlite3；避免原生模块编译 Python 自带，无需安装
计算引擎 Python 3.10 数值计算、GIS 距离计算、Excel 读写 内网已有 / embeddable 版（8MB）
GIS 计算 geopy（Python） Haversine 距离计算，轻量（1MB） whl 文件离线安装
Excel 读写 openpyxl（Python） 读取现有业务模板，输出规划方案表 whl 文件离线安装（5MB）
地图 Leaflet.js + 离线瓦片 轻量，支持离线地图包（mbtiles） 含在 node_modules；瓦片包单独带入
图表 ECharts 5 负荷曲线、评分雷达图 含在 node_modules
✅
无原生模块依赖。所有 npm 包均为纯 JavaScript，不包含 C++ 原生模块（已移除 better-sqlite3）。在外网 npm install 后，整个 node_modules 文件夹可以直接打包带入内网使用，不需要在内网重新安装。

# 1. 解压项目（node_modules 已在其中，不需要 npm install）
Expand-Archive ts-agent-full.zip -DestinationPath C:\work\

# 2. 验证 Node.js（版本必须与外网一致）

# 3. 验证 Python 标准库（无需安装，Python 内置）

# 4. 安装 Python 离线包（每个 whl 文件单独传入，单独安装）

# 5. 初始化数据库

echo {"db_path": "C:/ts-agent-data/ts_agent.db"} | python python\db_init.py

npm run dev   # 开发调试模式
