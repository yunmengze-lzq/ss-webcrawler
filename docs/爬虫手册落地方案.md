# 爬虫手册落地方案

## 目标

把内网取数流程从“写死脚本”改成“填写配置即可运行”：

1. 在业务系统中按 `F12 -> Network -> Fetch/XHR` 找到真实接口。
2. 复制接口 `URL`、`Headers/Cookie`、`Payload`。
3. 在工具里填写响应列表路径和字段映射。
4. 运行后自动保存 `raw.json`、`rows.json`、`rows.xlsx`。

## 为什么不默认建数据库

当前各系统数据格式还不稳定，字段名、分页方式、返回层级都需要先核对。如果一开始就建库，会很快遇到表结构反复改、历史数据难迁移的问题。

默认策略：

- `raw.json`：保留接口原始响应，方便追溯和重新解析。
- `rows.json`：保留字段映射后的标准行数据，方便智能体读取。
- `rows.xlsx`：给人工核对和比赛材料展示使用。
- SQLite：等字段口径稳定后，再把 `rows.json` 入库，给智能体做跨表检索、统计、预测和方案计算。

## 配置字段

| 字段 | 说明 | 示例 |
| --- | --- | --- |
| 配置名称 | 便于识别接口用途 | 计量负载率日数据 |
| 系统编号 | 数据来源系统 | load_meter |
| 请求方法 | GET 或 POST | POST |
| API URL | Fetch/XHR 完整接口地址 | http://10.x.x.x/api/query |
| Headers | 可粘贴 JSON，也可粘贴 `Key: Value` 多行格式 | Accept / Content-Type |
| Cookie | 从浏览器请求标头复制，保存在本机配置，不写入代码 | JSESSIONID=... |
| Payload JSON | 查询参数，日期、分页、台区编号等在这里改 | {"pageNo":1,"pageSize":100} |
| 列表路径 | 响应中数组所在路径 | data.list |
| 字段映射 JSON | 左侧是标准字段，右侧是接口字段路径 | {"ts_id":"stationId"} |

## 接口不通时的兜底

如果系统禁止直接访问接口，或真实数据必须点击页面按钮才能导出，仍然使用 RPA 发现模式：

1. 打开目标系统页面。
2. 人工完成查询条件。
3. 让工具捕获下载行为。
4. 保存 Excel/PDF 后再解析成标准 `rows.json`。

这类任务不强行走数据库，也不在内网下载任何依赖。
